Section 1 — Tokenization

Embeddings: Words in a Geometric World

  • A word embedding maps each token to a dense vector in ℝd — capturing meaning through geometry.
  • Similar words land near each other. "Cat" and "dog" cluster; "run" and "swim" cluster.
  • Vector arithmetic captures relationships: king − man + woman ≈ queen
  • Positional encoding adds absolute position via sin/cos functions — the model knows where each token is.

Section 2 — Core Mechanism

Self-Attention: Context Through Queries & Keys

  • Every token emits a query (what am I looking for?) and receives keys (what do I contain?) from all other tokens.
  • Attention score: score(q, k) = q · k / √dk
  • Softmax over all scores yields a weighted sum of values (v) — the attended output.
  • All in parallel: matrix operations Q = XWQ, K = XWK, V = XWV
Scaled dot-product prevents vanishing gradients in high-dimensional projections.

Section 3 — Parallel Attention

Multi-Head Attention

Run multiple attention heads in parallel, each with its own WQ, WK, WV — then concatenate and project.

01

Split into h heads

Each head operates on a reduced dimension dh = d / h, keeping total compute manageable.

02

Attend in parallel

Heads independently compute attention, capturing different relational patterns — syntax, coreference, semantics.

03

Concat & project

Output = WO · concat(head1, …, headh) — linear projection back to d.

Different heads learn different attention patterns — some focus on local context, others on distant dependencies.
Section 4 — Per-Layer Components

Feed-Forward & Residual Connections

  • FFN — two linear layers with GELU activation: FFN(x) = GELU(xW1)W2. Applies non-linearity to each token independently.
  • Residual connection: x ← x + Sublayer(x) — enables gradient flow and trains deep networks reliably.
  • LayerNorm: y = (x − μ) / √(σ² + ε) ⊙ γ + β — stabilizes training by normalizing activations per token.

Section 5 — Architecture

Encoder-Decoder Architecture

  • Encoder — N identical layers. Each has Multi-Head Self-Attention + FFN with residual & LayerNorm. Sees the full input sequence.
  • Decoder — N identical layers. Each has Masked Self-Attention (can't see future) + Encoder-Decoder Cross-Attention + FFN.
  • Cross-attention: queries come from decoder; keys & values come from encoder output — aligns decoding to source representation.
The original Transformer: 6 encoder layers, 6 decoder layers, dmodel=512.

Section 6 — Training

Training & Scaling

01 — Objective

Causal Language Modeling

Predict the next token given the prefix. Minimize cross-entropy loss over the training corpus. Teacher forcing during training.

02 — Optimization

AdamW + Warmup

AdamW with weight decay (decouples L2 from momentum). Learning rate warmup for warmup_steps then inverse-square-root decay.

03 — Scaling

Power Law Scaling

Loss ∝ N−0.07 · C−0.076 · D−0.057 (Kaplan et al.). Bigger models + more data + more compute = better performance.

04 — Key Insight

More Compute Wins

GPT-3 (175B params), PaLM (540B), Llama 3 (405B). The transformer architecture scales predictably with compute budget.
